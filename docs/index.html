CORE

What Is Ralph Loop?

A Bash while-loop that spawns fresh AI coding agents, each with zero context memory, until every user story in prd.json is verifiably complete. The loop is deterministic; the agent output is not — and that's the point.

The Problem

LLM context windows fill up with no free(). After ~70% utilization, agents enter the Dumb Zone — hallucinating, forgetting constraints, making worse decisions. Single-session agents degrade.

The Solution

Never let an agent enter the Dumb Zone. Every iteration spawns a fresh instance with clean context. The agent reads files to orient, implements one story, writes state back, and exits. The next iteration starts clean.

MECHANICS
How the Loop Works
STEP 1
Read State
Agent reads prd.json, progress.txt, AGENTS.md, and git log to orient itself.
cat prd.json
cat progress.txt
→ Picks highest-priority pending story
STEP 2
Implement
Implements exactly one user story. Frontend stories must pass browser-verified acceptance tests.
npm run dev
dev-browser verify
→ Verification passes
STEP 3
Commit State
Updates prd.json (passes: true), appends progress.txt, and git commits with the story ID.
git commit -m "feat(...)"
prd.json → passes:true
→ Output signal
STEP 4
Signal & Repeat
Outputs <promise>COMPLETE</promise>. The outer loop detects this, verifies prd.json, and spawns the next iteration.
grep "COMPLETE"
→ next iteration or exit
CAPABILITIES

Key Features

Everything an autonomous agent loop needs — and nothing it doesn't.

Fresh Context Per Iteration

Every coding agent starts with zero memory. No context rot, no "Dumb Zone" degradation. The subagent boundary is the context boundary.

File-Based State Machine

prd.json is the single source of truth. progress.txt is the cross-session handoff diary. git is the code snapshot. No database, no server.

Browser-Verified Acceptance

Frontend stories MUST pass browser automation tests (Puppeteer/Playwright). No "looks good to me" — verification is external and reproducible.

Smart Error Recovery

Two failed attempts → auto-revert, mark BLOCKED, spawn debug subagent. Never force a broken story.

Plugin Architecture

Install once as a Claude Code plugin. Reuse across projects. Skills, agents, hooks — all auto-discovered.

Dual Runtime

Bash for simplicity (ralph.sh) or Node.js for programmatic control (ralph-node.js). Hooks-based autonomous mode or CLI.

START

Quick Start

Three steps from zero to autonomous development.

  1. 1 Generate your PRD
    Describe your project. Ralph asks 5–10 questions, then generates prd.json with 50–200 fine-grained user stories — each small enough for one context window.
  2. 2 Initialize the project
    The initializer agent scaffolds your project: init.sh, AGENTS.md, progress.txt, and the initial git commit. No features implemented — just the foundation.
  3. 3 Run the loop
    The coordinator picks the highest-priority story, spawns a coding agent, verifies, commits, and repeats — until every story passes.
Install and run
# Install Ralph Loop as a Claude Code plugin claude plugin install ./ralph-loop --scope local # Initialize a project for Ralph claude -p "/ralph-loop:init" # Run the autonomous loop claude -p "/ralph-loop:run" # Or use the standalone CLI bin/ralph run --max-iterations 100 --verbose

# ADR 0001: Plugin Architecture — Separate Plugin from Target Project

Ralph Loop currently ships as a flat repo that gets copied into target projects. The decision is to split it into a proper Claude Code plugin (`ralph-loop-plugin/`) that gets installed once and reused across projects, with project-specific state files living in the target project.

## Why

**Hard to reverse.** Once we reorganize into `.claude-plugin/plugin.json` + `skills/` + `agents/` structure, all path references, invocation patterns, and the SKILL.md manifest change. Reverting would break installed instances.

**Surprising without context.** A future maintainer might expect everything in one repo like the current model. The split is not obvious — the plugin contains the engine and templates, but `prd.json`, `AGENTS.md`, and `progress.txt` are generated per target project. Without this ADR, someone might try to commit project state to the plugin repo.

**Real trade-off.** The alternative was keeping the flat repo (simple — one `git clone`, edit templates, run). The plugin model adds structure but enables: install-once-reuse-everywhere, proper versioning via `plugin.json`, skill namespacing (`/ralph-loop:run`), and auto-discovery by Claude Code. The cost is upfront restructuring complexity and a steeper initial setup (install plugin → init project → run vs. clone → edit → run).

## Decision

- The plugin repo contains: loop engine (`bin/`), skills, agents, hooks, templates, `.mcp.json`, and reference docs.
- The target project contains: `prd.json`, `AGENTS.md`, `progress.txt`, `init.sh` — generated by the Initializer Agent.
- The plugin is installed via `claude plugin install ./ralph-loop --scope local` (dev) or marketplace (distribution).
- Templates use `{{PLACEHOLDER}}` substitution from prd.json during initialization, with an optional `CLAUDE.custom.md` for user additions.

## Considered Alternatives

- **Keep flat repo**: Familiar workflow, no restructuring cost. Rejected because it doesn't scale across multiple target projects — every project needs a copy of the engine, and updates require manual sync.
- **NPM package**: Installable as `npx ralph-loop`. Rejected because it doesn't integrate with Claude Code's plugin discovery, hooks, or skill namespacing. The plugin model is the native extension mechanism.
